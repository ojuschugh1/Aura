# Configuration

On `aura init`, three config files are generated in `~/.aura/`:

---

## config.toml

Main daemon configuration.

```toml
[daemon]
port = 7437
log_level = "info"
shared_secret = "<auto-generated>"

[memory]
max_entries = 10000
max_size_mb = 100

[compression]
enabled = true
min_confidence = 0.8

[cost]
# Per-token pricing in USD
[cost.model_pricing]
"claude-sonnet" = 0.003
"gpt-4o" = 0.015

[trace]
ttl_days = 14
max_size_mb = 500

[subprocess]
timeout_seconds = 30
```

---

## policy.toml

Action approval rules. The policy engine evaluates every agent action and returns a disposition.

```toml
# Auto-approve read operations
[[rules]]
category = "read"
disposition = "auto-approve"

# Require approval for writes
[[rules]]
category = "write"
disposition = "require-approval"

# Deny network operations
[[rules]]
category = "network"
disposition = "deny"

# Override: auto-approve writes to test directory
[[overrides]]
category = "write"
disposition = "auto-approve"
path = "src/test/**"
```

**Hot-reload** — edit `policy.toml` while the daemon is running and changes take effect within 5 seconds.

Dispositions: `auto-approve`, `require-approval`, `deny`

---

## routing.toml

Model routing configuration. Routes tasks to the right model based on complexity.

```toml
[classification]
low_max_tokens = 500
medium_max_tokens = 2000

[models]
[models.low]
name = "gpt-4o-mini"
budget_usd = 5.00

[models.medium]
name = "claude-sonnet"
budget_usd = 20.00

[models.high]
name = "claude-opus"
budget_usd = 50.00
```

---

## Data directory

By default, all data is stored in `~/.aura/`. Override with `--dir`:

```bash
aura --dir /path/to/custom/dir memory ls
aura --dir /path/to/custom/dir wiki ingest design.md
```

This is useful for per-project wikis:

```bash
# Project-specific wiki
aura --dir ./.aura wiki ingest README.md
aura --dir ./.aura wiki query "architecture"
```
